## What This Project Does

This project applies Quantum Generative Adversarial Networks (QGANs) to real human sleep EEG recordings from the Sleep-EDF dataset, training a hybrid quantum-classical model to generate synthetic sleep data that matches the statistical distribution of real signals.

**Why this matters:** Real polysomnographic data is expensive and difficult to collect — patients must attend overnight lab studies. Synthetic EEG data generated by a QGAN can augment imbalanced training sets without additional patient recordings, addressing a core challenge in automated sleep staging.

**The quantum advantage confirmed:** Across all tested architectures and feature counts, the 6-qubit quantum generator (Arch C) consistently achieves lower standard deviation MAE than the classical baseline — a 46–71% reduction at 3 features — demonstrating that parameterized quantum circuits capture the variance structure of EEG signals with greater fidelity than classical MLPs of equivalent capacity.

---

## Key Findings

### 1. Quantum Variance Advantage
Both QGAN variants achieve substantially lower StdMAE than the Classical GAN across all feature configurations. At 3 features, QGAN Noiseless achieves StdMAE = 0.071 and QGAN Noisy achieves 0.064 — reductions of 67.7% and 70.9% over the classical baseline (0.220). This advantage persists under IBM 127-qubit hardware noise simulation.

### 2. Noise as Implicit Regularisation
Gaussian noise injection (σ = 0.1) during QGAN training functions as implicit regularisation at 3 features, yielding the best StdMAE of 0.064 across all nine experimental configurations (3 models × 3 feature counts) while maintaining competitive classification (Accuracy = 0.880, F1 = 0.880). This effect breaks down at 4 features, defining a feature-complexity boundary for noise-based regularisation.

### 3. Classical GAN Collapse Under Higher Dimensions
The Classical GAN degrades sharply at 3 and 4 features (Accuracy = 0.535 and 0.585), exhibiting mode-collapse behaviour under BCE loss. The WGAN-GP objective used in both quantum variants provides more stable training dynamics.

### 4. Entanglement Topology Controls Training Stability
All-to-all CNOT topology (Arch D) resolves discriminator collapse (Specificity: 0.000 → 0.880) under BCE loss alone — without requiring WGAN-GP. Circuit topology, not just loss function, determines whether adversarial training collapses.

### 5. Loss × Architecture Interaction is Non-Monotonic
WGAN-GP improves ring topology (+0.730 Specificity) but degrades all-to-all topology (−0.215 Specificity). These two design levers interact non-linearly — not predicted by existing quantum GAN theory.

## Architecture

All quantum models use **Arch C** as the final architecture: a 6-qubit parameterised quantum circuit paired with a classical MLP discriminator, trained under Wasserstein GAN with Gradient Penalty (λ = 10).

### Quantum Generator (Arch C)

```
Input: z ∈ R⁴ (random noise)

Step 1 — Encoding:
  Qubits 0–3 : RX(z_w) gates  →  encodes 4 input features
  Qubits 4–5 : RY(α) gates    →  learned ancilla bias parameters

Step 2 — Variational layers (L = 2):
  Ring CNOT: q0→q1→q2→q3→q4→q5→q0  (entanglement)
  RY(Θ_l,w) on all 6 qubits           (trainable rotation)

Output: ⟨Z_w⟩ for w = 0..3  (Pauli-Z expectation on feature qubits)
        Values in [−1, 1]
```

The 2 ancilla qubits expand the effective Hilbert space from 2⁴ = 16 to 2⁶ = 64 dimensions without enlarging the output dimensionality, enabling richer intermediate quantum state representations.

### Classical Discriminator (Wasserstein Critic)

```
Input → Linear(32) → LeakyReLU → Dropout(0.3) → Linear(16) → LeakyReLU → Linear(1)
No Sigmoid — raw Wasserstein scores, classification threshold at 0.0
```

Replacing the original quantum discriminator — which collapsed to Specificity = 0.000 due to barren plateau — with a classical MLP was the key architectural fix enabling stable WGAN-GP training.

### Architecture Ablation (4 Features, 50 Epochs)

| Architecture | Loss | Accuracy | Specificity | F1 | StdMAE | Time/ep |
|---|---|---|---|---|---|---|
| Arch A — Ring | BCE | 0.500 | 0.000 | 0.667 | **0.0797** | 67.6 s |
| Arch A — Ring | WGAN-GP | 0.825 | 0.730 | 0.840 | 0.0808 | 14.6 s |
| Arch B — RZ+Dual | BCE | 0.943 | 1.000 | 0.939 | 0.3076 | 17.4 s |
| Arch B — RZ+Dual | WGAN-GP | **0.995** | **1.000** | **0.995** | 0.3064 | 43.0 s |
| **Arch C — 6-Qubit** | **WGAN-GP** | **0.893** | **0.955** | **0.885** | **0.1051** | 252.1 s |
| Arch D — All-to-All | BCE | 0.920 | 0.880 | 0.923 | 0.1396 | 45.8 s |
| Arch D — All-to-All | WGAN-GP | 0.828 | 0.665 | 0.852 | 0.1364 | 173.2 s |

Arch C is the only configuration achieving both strong classification (Spec = 0.955) and competitive variance matching (StdMAE = 0.1051) simultaneously.

## How to Run

### CPU Experiments

```bash
# Noise experiment — 3 models × features 2/3/4 — 50 epochs
# Saves to results_noise.json and results_noise_2f/3f/4f.json
python -m qgan.train_noise

# Generate noise experiment figures and LaTeX tables
python -m qgan.plot_noise

# Architecture ablation — Arch B, C, D × BCE and WGAN-GP
python -m qgan.train_arch
python -m qgan.plot_arch

# Arch A baseline
python -m qgan.train        # BCE → results.json
python -m qgan.train_wgan   # WGAN-GP → results_wgan.json
python -m qgan.plot
python -m qgan.plot_wgan
```

### IBM QPU — Local Verification (3 Epochs)

```bash
# Set up IBM credentials first
python ibm_quantum_setup.py

# Local test — 3 epochs, features 2/3/4, verifies pipeline
python -m qgan.train_ibm
```

### IBM QPU — ARC Supercomputer (50 Epochs)

```bash
# On ARC after cloning repo and transferring credentials + data separately
python -m qgan.train_ibm --arc

# Recommended: SLURM submission
sbatch run_arc.sh
squeue -u $USER
tail -f qgan_<JOBID>.log

# Retrieve results
scp username@arc.edu:~/QGAN-Sleep-EEG/results_ibm_qpu*.json ./
```

> Transfer credentials and EDF file separately — never commit them:
> ```bash
> scp ibm_credentials.txt username@arc.edu:~/QGAN-Sleep-EEG/
> scp data/EPCTL03.edf    username@arc.edu:~/QGAN-Sleep-EEG/data/
> ```
