# 🧠 Quantum GAN for Sleep EEG Data Generation

A hybrid quantum-classical machine learning project applying **Quantum Generative Adversarial Networks (QGAN)** to generate synthetic sleep EEG data. Built with **PennyLane**, **PyTorch**, and **IBM Qiskit** for quantum noise simulation.

---

## 🚀 What This Project Does

This project takes real human sleep EEG brain signal recordings and trains a Quantum GAN to generate new synthetic sleep data that looks statistically realistic.

**Why this matters:** Real sleep data is expensive and hard to collect. Patients need to visit hospitals, wear electrodes, and sleep overnight in a lab. Synthetic sleep data generated by a QGAN can give researchers more training data without needing more patients.

**What makes this quantum:** The generator uses a parameterized quantum circuit. Quantum superposition and entanglement allow the generator to capture the statistical spread (variance) of EEG signals better than a classical neural network — confirmed experimentally across all tested architectures.

---

## ⚛️ How the QGAN Works

A Generator and a Discriminator compete against each other during training:

- **Quantum Generator** — takes random noise as input, encodes it into a quantum circuit using RX rotation gates, entangles qubits using ring CNOT gates, and outputs fake EEG feature vectors via Pauli-Z measurements.
- **Classical Discriminator** — a small MLP neural network (32→16→1) that scores whether its input looks real or fake. Classical discriminator replaces the original quantum discriminator which collapsed due to the barren plateau problem.

They compete until the Generator produces data the Discriminator cannot distinguish from real sleep data.

---

## 🔬 Architecture Variants Explored

Four quantum generator architectures were tested, each with both BCE and WGAN-GP loss functions:

| Architecture | Qubits | Entanglement | Gate Sequence | Key Property |
|---|---|---|---|---|
| **Arch A — Ring** (baseline) | 4 | Ring CNOT (0→1→2→3) | RX → CNOT → RY | Best variance matching |
| **Arch B — RZ+Dual** | 4 | Ring CNOT | RZ → RY → CNOT → RX | Most expressive gates |
| **Arch C — 6-Qubit** ⭐ | 6 | Ring CNOT | RX → CNOT → RY | Best overall balance |
| **Arch D — All-to-All** | 4 | All-pair CNOT | RX → CNOT → RY | Topology fixes collapse |

### ⭐ Final Recommended Architecture: Arch C + WGAN-GP

6-qubit quantum generator with classical MLP discriminator trained using Wasserstein GAN with Gradient Penalty. The only configuration that simultaneously achieves strong classification **and** preserves the quantum variance advantage.

## 🔑 Novel Findings

**1. Quantum Variance Advantage**
HybridQGAN achieves StdMAE = 0.0797 — 50% better than Classical GAN (0.1611). This advantage survives IBM hardware noise (StdMAE = 0.1149). All 6 quantum configurations beat the classical baseline in variance matching.

**2. Barren Plateau Confirmed on EEG**
Quantum discriminator collapses to Specificity = 0.000 across all feature counts. First empirical confirmation of discriminator barren plateau specifically on biomedical sleep EEG data.

**3. Entanglement Topology Controls Training Collapse**
All-to-all CNOT topology (Arch D) fixes Specificity = 0.000 → 0.880 using BCE loss alone — no WGAN-GP needed. Circuit topology, not just loss function, determines whether GAN training collapses.

**4. Loss × Architecture Interaction is Non-Monotonic**
WGAN-GP improves ring topology (+0.730 Specificity) but hurts all-to-all topology (−0.215 Specificity). The two design levers interact non-linearly — not predicted by existing theory.

**5. Quantum Generators are More Compatible with Wasserstein Training**
Classical GAN Specificity under WGAN-GP: 0.965 → 0.340 (degrades). HybridQGAN Specificity under WGAN-GP: 0.000 → 0.730–0.955 (fixed). The quantum circuit's constrained parameterization prevents the mode collapse that destroys classical WGAN-GP training.

**6. IBM Noise Does Not Eliminate Quantum Advantage**
QPU-Sim achieved StdMAE = 0.1149 — still better than Classical GAN (0.1611). Real quantum hardware training remains viable for biomedical EEG despite decoherence.

## 🖥️ How to Run on ARC Supercomputer (50 Epochs, Real QPU)

### What ARC is used for

All CPU experiments were run on a local machine. ARC is used only for the **real IBM QPU run** of Arch C + WGAN-GP across all 3 feature counts (2, 3, 4) with 50 epochs each — the final paper QPU results.

### Step 1 — Set up IBM credentials locally

```bash
# Edit ibm_quantum_setup.py with your IBM API key and CRN, then run:
python ibm_quantum_setup.py
# Creates: ibm_credentials.txt and ibm_backend.txt
```

> ⚠️ **Never push `ibm_credentials.txt` to GitHub.**

### Step 2 — Transfer files to ARC

```bash
# Clone project on ARC (code only — no credentials)
ssh username@arc.university.edu
git clone https://github.com/IkramAlgo/QGAN-Sleep-EEG.git

# Transfer credentials and data separately
scp ibm_credentials.txt username@arc.university.edu:~/QGAN-Sleep-EEG/
scp data/EPCTL03.edf    username@arc.university.edu:~/QGAN-Sleep-EEG/data/
```

### Step 3 — Set up environment on ARC

```bash
cd ~/QGAN-Sleep-EEG
conda create -n qgan310 python=3.10 -y
conda activate qgan310
pip install -r requirements.txt
```

### Step 4 — Test first (3 epochs, all features)

```bash
python -m qgan.train_ibm
```

Expected — if no errors across all 3 feature counts, code is ready:
```
EXPERIMENT: 2 features ['Mean', 'Std Dev']
  Epoch [  1/  3] G:... C:... MeanMAE:... StdMAE:... Time:...s
  ...
EXPERIMENT: 3 features ['Mean', 'Std Dev', 'Min']
  ...
EXPERIMENT: 4 features ['Mean', 'Std Dev', 'Min', 'Max']
  ...
Combined results saved: results_ibm_local.json
```

### Step 5 — Submit full QPU job (50 epochs, all features)

**Option A — Direct run:**
```bash
python -m qgan.train_ibm --arc
```

**Option B — SLURM submission (recommended):**

Create `run_arc.sh`:
```bash
#!/bin/bash
#SBATCH --job-name=qgan_arch_c
#SBATCH --output=qgan_%j.log
#SBATCH --error=qgan_%j.err
#SBATCH --time=48:00:00
#SBATCH --nodes=1
#SBATCH --ntasks=1
#SBATCH --cpus-per-task=8
#SBATCH --mem=32G

source ~/miniconda3/bin/activate qgan310
cd ~/QGAN-Sleep-EEG
python -m qgan.train_ibm --arc
```

```bash
sbatch run_arc.sh
squeue -u $USER
tail -f qgan_<JOBID>.log
```

### Step 6 — Retrieve results

```bash
scp username@arc.university.edu:~/QGAN-Sleep-EEG/results_ibm_qpu*.json ./
scp username@arc.university.edu:~/QGAN-Sleep-EEG/checkpoint_ibm_*.json ./
```
